# Ideas

## AI-assisted encoding (шаги 2 и 3)

### Контекст
В slow encode / skipped / encoding режимах пользователь проходит 4 шага запоминания слова.
Шаги 2 и 3 требуют творческих усилий и иногда вызывают ступор.

### Идея
Добавить кнопку "Подсказать" на шагах 2 и 3 — AI генерирует вариант как толчок, не как замену.

**Шаг 2 — звуковая ассоциация:**
Берём английское слово + транскрипцию → находим русское слово с похожим звучанием.
Пример: `"purple"` → звучит как "пёрпл" → предлагает "перепел", "пёрышко".

**Шаг 3 — сцена:**
Берём sourceText (русское слово) + soundAssociation (ассоциация из шага 2) → генерируем сцену где два образа взаимодействуют.
Пример: `"purple"` + "перепел" → *"Перепел сидит на троне в фиолетовой мантии"*.

### Важно
Своя ассоциация запоминается лучше. AI — только как разблокировка если совсем застрял, не как замена.

### Стек
- **Groq API** (бесплатный tier) — быстро, достаточно для этой задачи
- Модель: Llama 3 или Mixtral
- Реализация: Next.js Server Action → Groq API → варианты пользователю
- Fine-tuning не нужен, достаточно few-shot prompt с примерами
